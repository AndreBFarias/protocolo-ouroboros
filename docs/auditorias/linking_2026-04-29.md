## Hipóteses de causa raiz para subir cobertura ≥ 30% em valor

| # | Hipótese | Verificável via |
|---|---|---|
| H1 | Tipos sem entrada em `linking_config.yaml` caem em `default` (janela=3d, diff=2%, conf=0,75) — apertado para NFCe sem CNPJ canônico | `grep "^  [a-z_]*:" mappings/linking_config.yaml` cruzado com `tipos_documento.yaml` |
| H2 | DAS PARCSN usa âncora `vencimento` (Sprint 95b), mas o pagamento real costuma ser 1–10 dias antes/depois; janela 3d é insuficiente | testar com janela 14d em 14 DAS órfãos |
| H3 | NFCe modelo 65 paga em dinheiro (sem CNPJ contraparte) ou cartão (CNPJ é da bandeira, não do varejo), reduzindo score abaixo do confidence_minimo 0,85 | logs de scoring em runtime |
| H4 | Holerites 13º têm data_emissao em mês X mas pagamento em mês X+1 (depósito antecipado); janela 3d falha | testar com janela 14d em 4 holerites órfãos |
| H5 | Bancos C6, Santander, Nubank PJ não foram alvo do linking até hoje (talvez sprint passada filtrou só Nubank PF + Itaú) | `git log --grep linking` + grep filtros em código |
| H6 | Cobertura subiria 4–5× se janela_dias subisse para 14 e diff_valor_percentual relaxasse para 5% para tipos com `default` — provável cobertura pós-ajuste: 25–60% das 47 docs vinculados, mas só ~40 transações de 6.086 (0,7%) | sprint dedicada de tuning + reextração |

**Conclusão H**: ajuste de config sobe cobertura de **docs** (47 → ~40 vinculados), mas não escala cobertura de **transações** (sempre limitada a 47 documentos no universo total). **A escala real da Onda 4 só vem com mais ingestão documental: DOC-02 (mercado físico), DOC-19 (holerite com itens granulares), DOC-13 (multi-foto), DOC-14 (anti-duplicação semântica)**.

## Implicações táticas para Onda 4

1. **MICRO-01a é a sprint que pode subir a cobertura sem depender de DOC-02/DOC-19**: ela cria edges `transacao→nfce` e `nfce→item` para os 2 NFCe + 33 arestas `contem_item` que **já existem** no grafo. Se o linking atual não vincula nem 1 dos 2 NFCe a transação (cobertura NFCe = 0%), MICRO-01a precisa **(i) primeiro corrigir o linking documento↔transação para os 2 NFCe** OU (ii) trabalhar com vínculo manual via metadata de fornecedor.
2. **MICRO-03 demonstra a arquitetura mesmo com cobertura ínfima**. 2 transações com drill-down já validam o conceito; a aba "Cruzamento Micro" mostra honestamente a fração que tem item granular.
3. **IRPF-01 v1 precisa de DOIS banners** (não um): "investimentos não cobertos (DOC-20)" + "linking incompleto: ~13 de 6.086 transações vinculadas a documento (LINK-AUDIT-01 em curso)". Caso contrário usuário pensa que o ZIP é completo quando é parcial.
4. **GAP-01 vai ser potente** mas ruidoso: praticamente todas as 2.121 transações > R$ 100 caem como "gap documental" hoje. Limiar inicial deve ser alto (R$ 500 ou maior) para reduzir ruído inicial; refinar conforme cobertura cresce.
